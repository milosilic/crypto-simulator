# crypto-simulator

Static STACKBACK simulators (buyback analysis, DCA, lump sum, dynamic DCA) with bundled BTC price and precomputed daily risk scores. No build step — plain HTML, CSS, and JavaScript.

## Run locally

The simulators are static files, but the **DCA simulator** loads JSON via `fetch()`, so open it through a local HTTP server (not `file://`).

From the repo root:

```bash
python3 -m http.server 8889
```

Then open:

- [http://localhost:8889/](http://localhost:8889/) — index / navigation
- [http://localhost:8889/dca-simulator.html](http://localhost:8889/dca-simulator.html) — DCA, lump sum, dynamic DCA
- [http://localhost:8889/buyback-simulator.html](http://localhost:8889/buyback-simulator.html) — Binance trade buyback analysis (no JSON fetch; works offline once loaded)
- [http://localhost:8889/ytd-roi.html](http://localhost:8889/ytd-roi.html) — BTC year-to-date ROI by calendar year

Alternatives: `npx serve .`, VS Code Live Server, or any static file server.

**Requirements:** Node.js 18+ only for the data update scripts below. Browsers need network access for Tailwind/Chart.js CDNs on first load.

## Data files

| File | Description |
|------|-------------|
| `data/btc-usd-daily.json` | Daily BTC/USDT close from Binance (2017-08-17 → today) |
| `data/btc-risk-daily.json` | Daily risk score (0–1) derived from price vs power-law model |

## Update BTC prices

Refreshes `data/btc-usd-daily.json` from the Binance API (start date is fixed at 2017-08-17 in the script):

```bash
node scripts/fetch-btc-prices.mjs
```

Re-run this whenever you want newer daily closes. It overwrites the whole price file.

## Update risk scores

Risk scores are **not** computed in this repo. They are generated by the sibling project [**btc-risk-engine**](../btc-risk-engine) from the price JSON using a power-law model (see that repo’s `PLAN.md` for the spec).

Typical layout (both repos as siblings under the same parent directory):

```
projects/
  crypto-simulator/    ← this repo
  btc-risk-engine/     ← risk computation
```

**Full refresh workflow:**

```bash
# 1. Refresh prices (this repo)
node scripts/fetch-btc-prices.mjs

# 2. Recompute risk from prices (sibling repo)
cd ../btc-risk-engine
npm run export
```

`npm run export` reads `../crypto-simulator/data/btc-usd-daily.json` and writes `../crypto-simulator/data/btc-risk-daily.json`.

Custom paths:

```bash
node src/export-risk.mjs --sim-root /path/to/crypto-simulator
# or
node src/export-risk.mjs --prices /path/to/btc-usd-daily.json --out /path/to/btc-risk-daily.json
```

Environment variables: `BTC_PRICES_IN`, `BTC_RISK_OUT`.

After exporting, commit the updated JSON files in **crypto-simulator** only.

## Simulator notes

- **Lump sum / fixed DCA** — only needs `btc-usd-daily.json` (full range from 2017-08-17).
- **Dynamic DCA** — needs matching dates in both price and risk JSON. After extending prices backward, run `btc-risk-engine` export so risk covers the same dates.
